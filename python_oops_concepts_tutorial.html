1. What is OOP?

OOP stands for Object-Oriented Programming. It is a programming style where we organize code using classes and objects. Instead of writing everything as loose functions and variables, we group related data and behavior together.

In Python, OOP helps us model real-world things such as students, cars, employees, bank accounts, orders, and products. Each real-world thing can be represented as an object, and each object can have its own data and actions.

Real World Example ----------------- Car -> data : brand, model, color, speed -> action: start(), stop(), accelerate() In Python OOP: Class -> Blueprint Object -> Real item created from the blueprint
OOP makes code more reusable, easier to maintain, and closer to real-world thinking.

2. Class and Object

What is a Class?

A class is a blueprint or template. It defines what data and what behavior an object will have.

What is an Object?

An object is a real instance created from the class. If a class is a design of a house, the object is the actual built house.

class Student:
    def display(self):
        print("This is a student object")

s1 = Student()   # object creation
s1.display()

Explanation

  • class Student: creates a class named Student.
  • display(self) is an instance method.
  • s1 = Student() creates an object.
  • s1.display() calls the method using the object.

Example with Data

class Employee:
    def set_data(self, name, salary):
        self.name = name
        self.salary = salary

    def show_data(self):
        print("Name:", self.name)
        print("Salary:", self.salary)

e1 = Employee()
e1.set_data("Gopi", 50000)
e1.show_data()

Here, self.name and self.salary are object-specific variables. Different objects can store different values.

3. Constructor

A constructor is a special method that runs automatically when an object is created. In Python, the constructor method is __init__().

class Laptop:
    def __init__(self, brand, ram):
        self.brand = brand
        self.ram = ram

    def show(self):
        print("Brand:", self.brand)
        print("RAM:", self.ram, "GB")

l1 = Laptop("Dell", 16)
l2 = Laptop("HP", 8)

l1.show()
l2.show()

Why use constructor?

  • To initialize values at the time of object creation
  • To avoid calling a separate setup method later
  • To make object creation clean and professional
Object Creation Flow -------------------- Laptop("Dell", 16) | --> __init__(self, "Dell", 16) | --> self.brand = "Dell" --> self.ram = 16

4. Instance, Class, and Static Methods

Instance Method

Works with object data and uses self.

class Car:
    def __init__(self, brand):
        self.brand = brand

    def show_brand(self):
        print("Brand:", self.brand)

c1 = Car("Toyota")
c1.show_brand()

Class Variable and Class Method

Class variables are shared by all objects. Class methods use cls and can access class-level data.

class School:
    school_name = "ABC Public School"

    @classmethod
    def change_school_name(cls, new_name):
        cls.school_name = new_name

print(School.school_name)
School.change_school_name("XYZ International School")
print(School.school_name)

Static Method

A static method does not use self or cls. It is used for utility logic related to the class.

class MathHelper:
    @staticmethod
    def add(a, b):
        return a + b

print(MathHelper.add(10, 20))

5. Inheritance

Inheritance means one class can use the properties and methods of another class. The existing class is called the parent/base class, and the new class is called the child/derived class.

Animal (Parent) | +---- Dog (Child) | +---- Cat (Child)
class Animal:
    def eat(self):
        print("Animal eats food")

class Dog(Animal):
    def bark(self):
        print("Dog barks")

d1 = Dog()
d1.eat()
d1.bark()

Types of Inheritance in Python

  • Single Inheritance
  • Multilevel Inheritance
  • Multiple Inheritance
  • Hierarchical Inheritance

Multilevel Inheritance Example

class GrandParent:
    def house(self):
        print("Grandparent's house")

class Parent(GrandParent):
    def car(self):
        print("Parent's car")

class Child(Parent):
    def bike(self):
        print("Child's bike")

c = Child()
c.house()
c.car()
c.bike()

Multiple Inheritance Example

class Father:
    def skills(self):
        print("Father: Gardening")

class Mother:
    def talents(self):
        print("Mother: Painting")

class Child(Father, Mother):
    def own_skill(self):
        print("Child: Coding")

c1 = Child()
c1.skills()
c1.talents()
c1.own_skill()

6. Polymorphism

Polymorphism means one name, many forms. The same method name can behave differently depending on the object.

Method Overriding

class Bird:
    def sound(self):
        print("Bird makes a sound")

class Parrot(Bird):
    def sound(self):
        print("Parrot talks")

class Crow(Bird):
    def sound(self):
        print("Crow caws")

b1 = Parrot()
b2 = Crow()

b1.sound()
b2.sound()

Here, the child classes override the parent method.

Duck Typing in Python

Python also supports polymorphism through duck typing. If an object has the required method, Python can use it without checking its exact class type.

class PDFFile:
    def open(self):
        print("Opening PDF file")

class WordFile:
    def open(self):
        print("Opening Word file")

def start_file(file_obj):
    file_obj.open()

start_file(PDFFile())
start_file(WordFile())

7. Encapsulation

Encapsulation means wrapping data and methods into a single unit and controlling direct access to data. In Python, we usually use naming conventions and getter/setter methods for this purpose.

class BankAccount:
    def __init__(self, balance):
        self.__balance = balance   # private variable

    def deposit(self, amount):
        self.__balance += amount

    def get_balance(self):
        return self.__balance

acc = BankAccount(1000)
acc.deposit(500)
print("Balance:", acc.get_balance())
__balance is name-mangled. It should not be directly modified from outside the class in normal practice.

Why encapsulation?

  • Protects data from accidental misuse
  • Makes validation easier
  • Improves maintainability

8. Abstraction

Abstraction means hiding implementation details and showing only the necessary functionality. In Python, abstraction is commonly implemented using the abc module.

from abc import ABC, abstractmethod

class Vehicle(ABC):
    @abstractmethod
    def start(self):
        pass

class Bike(Vehicle):
    def start(self):
        print("Bike starts with self-start button")

class Car(Vehicle):
    def start(self):
        print("Car starts with key or button")

b = Bike()
c = Car()

b.start()
c.start()

The class Vehicle defines a common rule, but each child class gives its own implementation.

9. super() Keyword

The super() function is used to call methods or constructors from the parent class.

class Person:
    def __init__(self, name):
        self.name = name

class Teacher(Person):
    def __init__(self, name, subject):
        super().__init__(name)
        self.subject = subject

    def show(self):
        print("Name:", self.name)
        print("Subject:", self.subject)

t1 = Teacher("Meena", "Python")
t1.show()

This helps avoid rewriting parent initialization logic.

10. Special Methods (Magic Methods)

Python has special methods like __str__, __len__, and operator overloading methods. These help objects behave like built-in types.

class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __str__(self):
        return f"Book Title: {self.title}, Pages: {self.pages}"

b1 = Book("Python Basics", 250)
print(b1)

Operator Overloading Example

class Box:
    def __init__(self, value):
        self.value = value

    def __add__(self, other):
        return self.value + other.value

b1 = Box(10)
b2 = Box(20)
print(b1 + b2)

11. Mini Real-World Example: Employee Management

This example combines multiple OOP concepts in one program.

from abc import ABC, abstractmethod

class Person:
    company = "ItTechGenie"

    def __init__(self, name, age):
        self.name = name
        self.age = age

    def show_basic_info(self):
        print(f"Name: {self.name}, Age: {self.age}")

    @classmethod
    def show_company(cls):
        print("Company:", cls.company)

    @staticmethod
    def greet():
        print("Welcome to the Employee Portal")

class SalaryRule(ABC):
    @abstractmethod
    def calculate_salary(self):
        pass

class Employee(Person, SalaryRule):
    def __init__(self, name, age, emp_id, basic_salary):
        super().__init__(name, age)
        self.emp_id = emp_id
        self.__basic_salary = basic_salary

    def calculate_salary(self):
        hra = self.__basic_salary * 0.20
        da = self.__basic_salary * 0.10
        return self.__basic_salary + hra + da

    def show_employee(self):
        self.show_basic_info()
        print("Employee ID:", self.emp_id)
        print("Total Salary:", self.calculate_salary())

emp1 = Employee("Arun", 28, "E101", 30000)

Employee.greet()
Employee.show_company()
emp1.show_employee()

  • Constructor
  • Inheritance
  • Abstraction
  • Encapsulation
  • Class Method
  • Static Method

13. Interview Questions and Answers

What is the difference between a class and an object?

A class is a blueprint or template, while an object is a real instance created from that class. The class defines properties and methods, and the object holds actual values.

What is the purpose of the __init__ method?

The __init__ method is used to initialize object data when an object is created. It acts like a constructor and helps assign initial values to instance variables.

What is inheritance in Python?

Inheritance allows a child class to acquire the methods and properties of a parent class. It promotes code reuse and supports hierarchical program design.

What is polymorphism?

Polymorphism means one method name can behave differently for different objects. In Python, this is commonly achieved through method overriding and duck typing.

What is the difference between encapsulation and abstraction?

Encapsulation focuses on protecting and wrapping data within a class. Abstraction focuses on hiding internal implementation and exposing only the essential behavior.
