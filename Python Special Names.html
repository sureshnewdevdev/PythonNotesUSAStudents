1. What are special names or dunder names?

In Python, names such as __init__, __name__, __str__, and __len__ are called special names or dunder names.

Dunder means double underscore. These names begin and end with two underscores.

Examples of dunder names ------------------------ __init__ __name__ __main__ __str__ __repr__ __len__ __add__ __eq__ __iter__ __next__
Python uses these names internally for special behavior. When we define some of them in our classes, we are telling Python how our object should behave.

For example, by defining:

  • __init__ → we tell Python how to initialize an object
  • __str__ → we tell Python how to print an object
  • __len__ → we tell Python what to return for len(object)
  • __add__ → we tell Python how + should work for our objects

2. Meaning of __init__ and __name__

2.1 What is __init__?

__init__ is a special method used in a class. It runs automatically when an object is created.

class Student:
    def __init__(self, name):
        self.name = name
        print("Object created successfully")

s1 = Student("Gopi")
print(s1.name)
Meaning: __init__ initializes the object. It is commonly called the constructor in Python.

When you write s1 = Student("Gopi"), Python automatically calls the __init__() method.

Object Creation Flow -------------------- Student("Gopi") | ---> __init__(self, "Gopi") | ---> self.name = "Gopi"

2.2 What is __name__?

__name__ is a special built-in variable in Python. It tells you how the current Python file is being executed.

print(__name__)

It can have two common situations:

  • If the file runs directly, __name__ becomes "__main__"
  • If the file is imported into another file, __name__ becomes the module name
if __name__ == "__main__":
    print("This file is running directly")
else:
    print("This file is imported from another file")
Meaning: __name__ helps Python identify the execution mode of the file.

2.3 Difference between __init__ and __name__

Feature __init__ __name__
Type Special method Built-in variable
Used in Inside a class In a Python file/module
Purpose Initialize object data Tell how file is executed
Runs when Object is created Available automatically during execution

3. What is __main__?

__main__ is not a function. It is a special string value that Python assigns to __name__ when the file is run directly.

if __name__ == "__main__":
    print("Program started directly")
This is very useful when you want some code to run only when the file is executed directly, not when it is imported into another file.

Example

class Demo:
    def __init__(self):
        print("Demo object created")

def test():
    print("This is a test function")

if __name__ == "__main__":
    d = Demo()
    test()

If this file is imported into another Python file, the object creation and test call inside the if block will not run automatically.

4. Other important special names with explanations

4.1 __str__

Used to define how an object should look when printed in a user-friendly way.

class Book:
    def __init__(self, title):
        self.title = title

    def __str__(self):
        return f"Book Title: {self.title}"

b1 = Book("Python Basics")
print(b1)

Without __str__, Python prints an object in a default memory-style format.

4.2 __repr__

Used to define the developer-friendly or debugging representation of an object.

class Car:
    def __init__(self, brand):
        self.brand = brand

    def __repr__(self):
        return f"Car('{self.brand}')"

c1 = Car("Toyota")
print(c1)

4.3 __len__

Used when len() is called on an object.

class Team:
    def __init__(self, members):
        self.members = members

    def __len__(self):
        return len(self.members)

t = Team(["A", "B", "C"])
print(len(t))

4.4 __del__

Called when an object is about to be destroyed.

class Demo:
    def __del__(self):
        print("Object is destroyed")

d = Demo()
del d
Do not depend heavily on __del__ for critical work because object cleanup timing may vary.

4.5 __doc__

Stores the documentation string of a class, function, or module.

class Employee:
    """This class stores employee details."""
    pass

print(Employee.__doc__)

4.6 __dict__

Stores the object’s attributes in dictionary form.

class User:
    def __init__(self, name, age):
        self.name = name
        self.age = age

u1 = User("Gopi", 25)
print(u1.__dict__)

4.7 __class__

Tells which class the object belongs to.

class Animal:
    pass

a = Animal()
print(a.__class__)

4.8 __module__

Shows the module name in which the class was defined.

class Sample:
    pass

s = Sample()
print(s.__module__)

4.9 __bases__

Shows the parent classes of a class.

class A:
    pass

class B(A):
    pass

print(B.__bases__)

4.10 __add__

Used when the + operator is applied to objects. This is operator overloading.

class Number:
    def __init__(self, value):
        self.value = value

    def __add__(self, other):
        return self.value + other.value

n1 = Number(10)
n2 = Number(20)
print(n1 + n2)

4.11 __sub__, __mul__, __truediv__

Used for -, *, and / operators.

class Number:
    def __init__(self, value):
        self.value = value

    def __sub__(self, other):
        return self.value - other.value

    def __mul__(self, other):
        return self.value * other.value

    def __truediv__(self, other):
        return self.value / other.value

n1 = Number(20)
n2 = Number(5)

print(n1 - n2)
print(n1 * n2)
print(n1 / n2)

4.12 __eq__, __lt__, __gt__

Used for comparison operations like ==, <, and >.

class Product:
    def __init__(self, price):
        self.price = price

    def __eq__(self, other):
        return self.price == other.price

    def __lt__(self, other):
        return self.price < other.price

    def __gt__(self, other):
        return self.price > other.price

p1 = Product(500)
p2 = Product(700)

print(p1 == p2)
print(p1 < p2)
print(p2 > p1)

4.13 __getitem__ and __setitem__

These are used when accessing and assigning values using square brackets.

class MyList:
    def __init__(self, data):
        self.data = data

    def __getitem__(self, index):
        return self.data[index]

    def __setitem__(self, index, value):
        self.data[index] = value

obj = MyList([10, 20, 30])
print(obj[1])

obj[1] = 99
print(obj.data)

4.14 __contains__

Used for the in keyword.

class MyCollection:
    def __init__(self, items):
        self.items = items

    def __contains__(self, item):
        return item in self.items

c = MyCollection([1, 2, 3, 4])
print(3 in c)

4.15 __call__

Makes an object callable like a function.

class Greeting:
    def __call__(self, name):
        print("Hello", name)

g = Greeting()
g("Gopi")

4.16 __iter__ and __next__

Used for iteration and loops.

class Counter:
    def __init__(self, limit):
        self.current = 1
        self.limit = limit

    def __iter__(self):
        return self

    def __next__(self):
        if self.current <= self.limit:
            value = self.current
            self.current += 1
            return value
        else:
            raise StopIteration

c = Counter(3)

for i in c:
    print(i)

5. Classification table

Program/module info __name__, __main__, __module__, __doc__ Give information about the file or class definition
Object information __dict__, __class__, __bases__ Inspect object or class structure
Display __str__, __repr__ Control text display of objects
Length and collections __len__, __getitem__, __setitem__, __contains__ Make objects behave like collections
Operators __add__, __sub__, __mul__, __truediv__, __eq__, __lt__, __gt__ Operator overloading and comparisons

6. Practical combined examples

6.1 Combining __init__, __str__, and __len__

class Course:
    def __init__(self, title, students):
        self.title = title
        self.students = students

    def __str__(self):
        return f"Course: {self.title}"

    def __len__(self):
        return len(self.students)

c1 = Course("Python OOP", ["Arun", "Meena", "Rahul"])
print(c1)
print(len(c1))

6.2 Using __name__ == "__main__"

class Employee:
    def __init__(self, name):
        self.name = name

    def show(self):
        print("Employee Name:", self.name)

def run_demo():
    e1 = Employee("Gopi")
    e1.show()

if __name__ == "__main__":
    run_demo()
In this example, __init__ initializes the object and __name__ == "__main__" controls when the demo should run.

8. Interview questions and answers

What is the difference between __init__ and __name__?

__init__ is a special method inside a class and is used to initialize object data when an object is created. __name__ is a built-in variable that tells how the current Python file is being executed.

Why do we use if __name__ == "__main__": in Python?

We use it to make sure that a block of code runs only when the file is executed directly. If the file is imported into another program, that block will not run automatically.

What is the use of __str__ in Python?

__str__ is used to return a readable string representation of an object. It improves the output when the object is printed.

What is operator overloading in Python?

Operator overloading means defining how operators such as +, -, or == should behave for user-defined objects. This is done using methods such as __add__ and __eq__.

What are dunder methods?

Dunder methods are special methods that start and end with double underscores. Python calls them automatically in specific situations to provide built-in behavior to objects.
